# PRD — 2026 북중미 월드컵 대한민국 정보·홍보 웹페이지

## 1. 개요 (Overview)

| 항목 | 내용 |
|------|------|
| **프로젝트명** | 2026 북중미 월드컵 with 대한민국 |
| **목적** | 2026 FIFA 월드컵을 대한민국 경기 위주로 소개하는 **정보성 + 홍보성** 웹페이지 제작 |
| **타깃 사용자** | 대한민국 대표팀을 응원하는 국내 축구 팬, 월드컵 입문자 |
| **핵심 가치** | ① 한눈에 보는 대한민국 경기 정보 ② 깊이 있는 조별·선수 분석 ③ 2026 대회의 새로움 강조 |
| **플랫폼** | 반응형 웹 (모바일 우선) |

### 핵심 콘텐츠 4축
1. 대한민국 경기 일정 & 정보
2. 대한민국이 속한 **A조** 분석
3. A조 각 팀 주요 선수 분석
4. 2026 월드컵이 이전 대회와 달라진 점

### ✅ 확정 데이터 — 대한민국 A조 (웹 검색 확인)
조별 경기 기간: **2026년 6월 11일 ~ 6월 24일**

| 팀 | 비고 |
|----|------|
| 🇲🇽 멕시코 | 개최국(공동), 조 1번 시드 |
| 🇰🇷 대한민국 | 손흥민 중심, 16강 도전 |
| 🇿🇦 남아프리카공화국 | 아프리카 대표 |
| 🇨🇿 체코 | 유럽 대표 |

> **데이터 정책**: 위 조 편성은 확정. 세부 경기 일정·경기장·최종 선수 명단은 공식 소스(FIFA, 대한축구협회 KFA)에서 확정 후 반영한다. 코드는 데이터와 분리(예: JSON)하여, 데이터만 교체하면 화면이 갱신되도록 설계한다.

---

## 2. 단계별 개발 계획 (4 Phases)

### 🟢 Phase 1 — 기초 설계 & 메인 랜딩 페이지
**목표:** 프로젝트 뼈대와 첫인상(홍보성) 완성

- **기술 스택: React + Vite (확정)**
  - 팀/선수 카드를 컴포넌트로 재사용하기 좋고, 데이터(JSON)와 화면 분리에 유리
  - 폴더 구조, 디자인 토큰(색상: 태극기 레드/블루 기반, 폰트) 정의
- **메인 히어로 섹션**
  - 대회 슬로건, 개최지(미국·캐나다·멕시코), 대회 기간 표시
  - 대회 개막 / 대한민국 첫 경기까지 **D-Day 카운트다운**
- **전역 네비게이션** (경기일정 / 조분석 / 선수분석 / 2026 변경점)
- **데이터 구조 설계**: `teams.json`, `players.json`, `matches.json` 스키마 정의

**완료 기준:** 메인 페이지가 반응형으로 뜨고, 네비게이션으로 빈 섹션 이동 가능

---

### 🟡 Phase 2 — 대한민국 경기 & 조별 분석
**목표:** 프로젝트의 핵심 정보 콘텐츠 구축

- **대한민국 경기 일정 섹션**
  - 경기별 카드: 상대팀, 날짜·시간(한국시간 KST 변환), 경기장
  - 경기 결과 표시 영역(대회 진행 중 업데이트 대비)
- **조(Group) 분석 섹션**
  - 조 편성표(팀, FIFA 랭킹, 국기)
  - 조 전체 전력 분석 / 대한민국의 진출 시나리오 코멘터리
  - 상대 전적, 조 난이도 평가

**완료 기준:** 대한민국 경기와 소속 조 정보가 데이터 기반으로 정확히 렌더링

---

### 🟠 Phase 3 — 팀별 주요 선수 분석
**목표:** 깊이 있는 분석 콘텐츠로 체류시간 강화

- **팀 상세 페이지/모달**: 조의 각 팀 클릭 시 진입
  - 팀 소개, 감독, 최근 폼, 예상 포메이션
- **주요 선수 카드** — 표시 정보(확장 반영):
  - 포지션, 등번호, **나이, 키, 소속팀/리그, A매치 출전, A매치 골**
  - 핵심 특징 코멘트, "주의해야 할 선수(watch)" 하이라이트, 주장(captain) 표시
- **선수 선정 기준**: 유럽 5대리그 (준)주전은 모두 포함, 없으면 팀당 최소 3명(다출전·주장·특징 기준)
- **대한민국 선수단 별도 강조 섹션** (홍보성 강화)
- **"흥미로운 이야기" 별도 섹션**: 줄글 카드 형태, 항목을 계속 추가 가능 (`funfacts.json`)
- **경기 일정 카드에 경기장 정보 보강**: 나라·도시·해발고도 (Phase 2 섹션 확장)

**완료 기준:** 조 내 모든 팀의 주요 선수 정보 열람 가능, 한국 선수 강조 노출

> **데이터 추가 반영(스키마)**: `players.json`에 age/height/caps/goals/club/league/captain/**photo** 필드 추가, `venues.json`(경기장 나라·도시·해발고도) 신설, `funfacts.json`(흥미로운 이야기) 신설, `othergroups.json`(다른 조 화제거리) 신설.

> **추가 기능(Phase 3 확장)**:
> - **선수 사진**: 카드에 사진 영역. `photo`에 경로/URL 넣으면 표시, 비우면 등번호 아바타로 대체. 로컬 이미지는 `public/players/`.
> - **종료 경기 하이라이트/기사**: `matches.json`의 `links`(type: highlight|article)를 종료된 경기 카드에 버튼으로 노출.
> - **다른 조 화제거리 섹션**: A조 외 다른 조의 스토리라인 카드 (세로 섹션, 네비 '다른 조').
>
> **레이아웃 원칙**: 큰 구조는 세로 스크롤(홍보·SEO·공유 유리), 세부 전환만 섹션 내 탭 사용(예: 선수 팀 탭).
>
> **최종 섹션 순서(IA)**: 히어로 → ① 2026 변경점(48개국·32강 = 판 깔기) → ② 경기 일정 → ③ A조 분석 → ④ 선수 분석 → ⑤ 이야기 → ⑥ 다른 조(시야 확장). 변경점을 앞에 둬 32강 개념을 먼저 이해하게 하고, A조 콘텐츠를 한 덩어리로 연속 배치.

---

### 🔵 Phase 4 — 2026 변경점 & 홍보 마감 + 배포
**목표:** 대회의 새로움 전달, 완성도·공유성 마무리

- **2026 월드컵 변경점 섹션**
  - 본선 32 → **48개국 확대**, 3개국 공동 개최, 조 편성·토너먼트 방식 변화 등
  - 인포그래픽/비교 표로 시각화
- **홍보성 요소**
  - 응원 메시지, SNS 공유 버튼, 명장면/하이라이트 임베드(선택)
- **마무리 작업**
  - 반응형 QA, 접근성(대비·alt), 메타태그/OG 태그(공유 미리보기)
  - 성능 최적화 후 배포(GitHub Pages / Vercel / Netlify)

**완료 기준:** 4개 핵심 콘텐츠 완성, 모바일·PC QA 통과, 실제 URL 배포

---

## 3. 비기능 요구사항 (Non-Functional)
- **반응형**: 모바일 우선, 360px~데스크탑 대응
- **접근성**: 색 대비 준수, 이미지 alt, 키보드 내비게이션
- **성능**: 이미지 최적화, 첫 로딩 가벼움
- **유지보수**: 데이터(JSON)와 화면(컴포넌트) 분리

## 4. 향후 확장 아이디어 (Out of Scope, 추후)
- 실시간 스코어 API 연동
- 토너먼트 대진표 인터랙티브 뷰
- 다국어(영어) 지원
- 사용자 응원 댓글/투표
